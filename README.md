# Бот MAX: АЗС Долгоруковского округа

Сайт не требуется. GitHub Actions запускает скрипт примерно раз в 10 минут, открывает публичную карту `sberazs.ru`, ищет сведения об АЗС в границах Долгоруковского округа и отправляет сводку в канал MAX.

## Что понадобится

1. Верифицированный профиль организации, ИП или самозанятого на платформе MAX для партнёров.
2. Созданный и прошедший модерацию бот MAX с токеном.
3. Публичный канал MAX, куда бот добавлен администратором с правом публикации.
4. Аккаунт GitHub.

## 1. Создание бота MAX

На платформе MAX для партнёров откройте `Чат-боты → Создать`, заполните карточку и отправьте бота на модерацию. После одобрения откройте `Чат-боты → Перейти → Расширенные настройки → Настроить` и скопируйте токен.

Токен нельзя публиковать, вставлять в файлы проекта или отправлять посторонним.

## 2. Добавление бота в канал

В MAX откройте канал, нажмите его название, затем `Администраторы → Добавить администратора`. Найдите созданного бота, добавьте его и разрешите публикацию сообщений.

Канал должен иметь публичную ссылку, например:

`https://max.ru/selskiezori48`

Для настройки понадобится только часть после `max.ru/`:

`selskiezori48`

## 3. Создание репозитория GitHub

1. Распакуйте ZIP-архив.
2. На GitHub нажмите `+ → New repository`.
3. Назовите репозиторий, например `dolgorukovo-max-fuel-bot`.
4. Выберите `Public`.
5. Нажмите `Create repository`.
6. Нажмите `Add file → Upload files`.
7. Перетащите всё содержимое распакованной папки, включая `.github`.
8. Нажмите `Commit changes`.

После загрузки на главной странице должны быть видны `package.json`, `README.md`, папка `scripts`, а внутри `.github/workflows` — файл `fuel-bot.yml`.

## 4. Добавление секретов

Откройте репозиторий:

`Settings → Secrets and variables → Actions → New repository secret`

Создайте два секрета:

- `MAX_BOT_TOKEN` — полный токен бота MAX;
- `MAX_CHANNEL_LINK` — публичная ссылка канала без `https://max.ru/` и без `@`, например `selskiezori48`.

`MAX_TARGET_CHAT_ID` создавать не требуется, если канал публичный.

## 5. Отдельная проверка MAX

1. Откройте вкладку `Actions`.
2. Слева выберите `Проверка АЗС и публикация в MAX`.
3. Нажмите `Run workflow`.
4. В поле `Что проверить` выберите `test-max`.
5. Ещё раз нажмите `Run workflow`.

В канал должно прийти сообщение: `Проверка MAX-бота прошла успешно`.

Если этот тест не прошёл, проблема относится к токену, ссылке канала или правам бота, а не к сайту Сбера.

## 6. Проверка получения данных АЗС

Повторите ручной запуск, но выберите режим `fuel-check`.

Скрипт откроет `sberazs.ru`, попробует найти публичные сетевые ответы карты, отберёт объекты в границах Долгоруковского округа и отправит сводку в канал.

Если данные не распознаны, скрипт ничего не опубликует. В завершённом запуске появится диагностический архив `fuel-bot-diagnostics`.

## 7. Автоматическая работа

После загрузки workflow расписание включается автоматически. Запуск запланирован примерно каждые 10 минут. GitHub может иногда задерживать задание на несколько минут.

Текущая версия отправляет сводку при каждом успешном запуске. То есть при постоянной работе в канал может приходить до 144 сообщений в сутки. Для публикации только при изменениях требуется отдельное постоянное хранилище предыдущего состояния.

## Ограничения

- У `sberazs.ru` нет публично документированного API. Код анализирует только данные, которые публично загружает сама страница. При изменении сайта парсер может перестать работать.
- Нельзя обходить авторизацию, CAPTCHA и техническую защиту источника.
- GitHub Actions не гарантирует запуск секунда в секунду.
- В публичных репозиториях scheduled-workflow может отключиться после 60 дней отсутствия активности в репозитории.
